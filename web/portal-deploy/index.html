Hydrax / Workflow Platform

Portals

White-label institutional workflow platform above HydraX's regulated tokenisation, trading, and custody rails. Five role-aware portals share one design system.

Demo mode. All five portals are running with canned fixtures (VITE_DEMO_MODE=true); no live BFF backend. Mutations don't round-trip. Real bootstrap pairs slice 2b magic-link with slice 2c SMTP transport — see docs/architecture.md in the repo.

Issuer

issuer-portal

Tokenized-product workbench — create, approve, transition lifecycle states.

Distributor

distributor-portal

Approval queue + decision UI for product launch and subscription approvals.

Investor

investor-portal

Subscription lookup + upstream-health visibility into the platform.

Ops

ops-console

Audit-event search; SLA + incident surface placeholders for ops teams.

Admin

admin

Tenant + user + role + integrations management for platform administrators.
Source: github.com/naimkatiman/hydrax-app. Each portal is a Vite + React + Redux Toolkit static bundle. Shared primitives live in web/packages/ui + web/packages/tenant-theme.
